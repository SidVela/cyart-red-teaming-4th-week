# Cloud Environment Attacks

## Objective

Understand common cloud security risks, misconfigurations, and defensive monitoring techniques in cloud environments.

## Core Concepts

### Cloud Reconnaissance

Cloud reconnaissance involves identifying cloud resources, services, and configurations that may expose security weaknesses.

Examples:

- Public storage buckets
- Exposed services
- Excessive permissions

### Privilege Escalation

Improper Identity and Access Management (IAM) permissions can allow users or services to gain elevated privileges.

### Data Exposure

Sensitive data may become accessible through:

- Misconfigured storage
- Weak access controls
- Shared credentials

## Cloud Platforms

- Amazon Web Services (AWS)
- Microsoft Azure
- Google Cloud Platform (GCP)

## MITRE ATT&CK References

- T1580 – Cloud Infrastructure Discovery
- T1078.004 – Valid Accounts: Cloud Accounts
- T1537 – Transfer Data to Cloud Account

## Security Best Practices

- Principle of Least Privilege
- Multi-Factor Authentication (MFA)
- Continuous Monitoring
- Security Auditing
- Access Reviews

## Learning Outcome

Understand cloud attack paths and the defensive controls used to secure cloud environments.
