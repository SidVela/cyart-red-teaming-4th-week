# Advanced Command and Control (C2) Frameworks

## Objective

Understand how Command and Control (C2) frameworks are used in controlled security testing environments to manage simulated endpoints and evaluate defensive monitoring.

## Core Concepts

### C2 Architecture

A Command and Control framework enables centralized communication between a security testing platform and controlled lab systems.

### Session Management

C2 platforms allow operators to track and manage active sessions during testing exercises.

### Communication Channels

Common communication methods include:

- HTTPS
- DNS
- TCP
- SMB

These channels are commonly analyzed by defenders for detection opportunities.

### MITRE ATT&CK References

- T1071 – Application Layer Protocol
- T1105 – Ingress Tool Transfer
- T1573 – Encrypted Channel

## Learning Outcome

Understand the role of C2 infrastructure in adversary emulation and defensive detection testing.
